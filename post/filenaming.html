Konvensi Penamaan File & Folder; Best Practices

Sangatlah penting untuk konsisten dan deskriptif dalam penamaan file. Nama file yang deskriptif akan membantu pengunjung dan pengguna saat menyimpan file.

  1. Nama file harus deskriptif tentang file tersebut. Pengunjung mungkin akan menyimpannya.
  2. Jangan gunakan spasi. Sebagai gantinya gunakan:
    • tanda hubung: nama_file.doc (lebih disukai untuk mesin pencari)
    • atau garis bawah: nama_file.doc
    • atau tanda Camel Case: Namafile.doc
  3. Karakter khusus harus dihindari: ~ ! @ # $ % ^ & * ( ) ` ; < > ? , { } ' "
  4. Hanya 1 titik per nama file dan berada tepat sebelum ekstensi file
  5. Jika tanggal disertakan, selalu gunakan tanggal 8 digit numerik dalam format YYYYMMDD.
  6. Jika hanya bulan dan tahun yang tersedia: YYYYMM
  7. Jika hanya tahun yang tersedia: YYYY
  8. Tambahkan tahun di depan nama file jika penting untuk **mengurutkan berdasarkan tahun.
  9. Tambahkan tanggal di akhir nama file jika penting untuk mengurutkan berdasarkan nama
  10. Jangan membuat nama file terlalu panjang; nama yang terlalu panjang tidak dapat digunakan dengan baik pada semua jenis perangkat lunak
  11. Untuk penomoran berurutan, **gunakan angka nol di depan** untuk memastikan file diurutkan dengan benar. "001, 002", bukan "1, 2, dst.

Pertimbangkan halal berikut saat menamai file Anda

  • Nama proyek
  • Informasi lokasi
  • Nama atau inisial peneliti
  • Tanggal atau rentang tanggal
  • Jenis data
  • Keterbacaan
